# 使用指南

## 适用场景

这个项目适合在 SJTU 课程播放页里做三件事：

- 下载课程音频
- 转录课程字幕
- 生成 AI 笔记

任务以后端服务为准。只要本地服务还在运行，前端页面刷新、切页、从课程 A 切到课程 B，都不会中断已经入队的任务。

## 启动步骤

### 1. 安装依赖

```bash
uv sync
```

### 2. 配置环境

```bash
cp env.example .env
```

最少需要配置：

```bash
GOOGLE_API_KEY=your_api_key_here
```

如果你希望自定义输出目录，也可以额外配置：

```bash
DOWNLOAD_DIR=/path/to/downloads
OBSIDIAN_VAULT_PATH=/path/to/notes
TEMP_DIR=/path/to/temp
```

### 3. 启动服务

```bash
uv run python auto_study_server.py
```

### 4. 安装浏览器脚本

1. 安装 Tampermonkey
2. 新建脚本并粘贴 `sjtu_video_helper.js`
3. 保存后进入 SJTU 课程播放页

## 面板行为

当前版本只允许真正的深层播放页持有 `Notes Helper` 面板。

- 顶层 Canvas 壳页面不会再显示浅层错误面板
- 非播放页不会提供新增任务按钮
- 已提交任务会从后端恢复并继续显示

## 常见操作

### 生成 AI 笔记

1. 打开课程视频页
2. 播放视频，等待脚本检测到视频源
3. 点击 `生成 AI 笔记`

服务端会自动补齐缺失步骤：

- 缺音频时先下载
- 缺字幕时再转录
- 最后生成 Markdown 笔记

### 仅下载音频

1. 播放视频
2. 点击 `下载音频`
3. 任务进入后端队列

### 仅转录

1. 进入课程视频页
2. 点击 `转录音频`
3. 如果音频已存在，任务会直接使用现有音频

### 下载视频

点击 `下载视频` 后，服务端会按视频模式输出 `.mp4`，不再错误地落到音频下载流程。

## 任务模型

- 后端默认串行执行
- 前端只是状态显示和任务提交入口
- 重复任务允许继续排队
- 任务真正执行时，会根据当前文件状态决定跳过还是补做
- 如果用户想强制重跑，应显式确认覆盖

## 输出目录

默认输出目录位于仓库内：

- `data/downloads`
- `data/notes`
- `data/temp`

如果配置了环境变量，则以环境变量为准。

## 故障排查

### 看不到面板

- 确认当前页面是实际播放页，而不是课程列表页或 `external_tools` 宿主页
- 确认 Tampermonkey 脚本已启用
- 打开控制台检查是否有 `Notes Helper` 初始化日志

### 面板在，但没有视频源

- 先播放视频，再等待 2-3 秒
- 某些页面的视频元素是延迟挂载的

### 无法连接后端

- 确认 `uv run python auto_study_server.py` 正在运行
- 确认本地 `5000` 端口没有被其他服务占用

### 任务恢复失败

- 刷新页面后，面板会从 `/tasks` 恢复任务
- 如果后端服务重启，内存中的历史任务会丢失

## 建议

- 开启默认串行执行，避免显卡被多份转录同时抢占
- 公开仓库时不要提交 `.env`、下载内容和本地模型目录
